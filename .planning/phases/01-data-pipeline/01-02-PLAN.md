Replace it entirely with the following complete implementation:

```bash
          # Extract reference URLs (null-safe — handles reserved/rejected CVEs where
          # .containers.cna or .references may be absent)
          URLS=$(echo "$RESPONSE" | jq -r '
            (.containers.cna.references // [])
            | .[].url
          ' 2>/dev/null || true)

          # Count non-empty lines
          REF_COUNT=$(printf '%s' "$URLS" | grep -c . 2>/dev/null || echo "0")

          # Empty reference guard: GHA treats fromJSON([]) as a workflow-level error.
          # Must check count here and skip matrix output if zero — cannot be caught downstream.
          if [ "$REF_COUNT" -eq 0 ]; then
            echo "has_refs=false" >> "$GITHUB_OUTPUT"
            echo "matrix=[]" >> "$GITHUB_OUTPUT"
            echo "ref_count=0" >> "$GITHUB_OUTPUT"
            {
              echo "### ${CVE_ID}: No References Found"
              echo ""
              echo "This CVE has no reference URLs. Nothing to archive."
            } >> "$GITHUB_STEP_SUMMARY"
            exit 0
          fi

          # Build fromJSON-compatible matrix: array of {index, url} objects.
          # Index is embedded here (not derived from URL) to avoid artifact name collisions
          # and because strategy.job-index is not reliably accessible inside matrix steps.
          MATRIX=$(printf '%s' "$URLS" | jq -R -s '
            split("\n")
            | map(select(length > 0))
            | to_entries
            | map({"index": .key, "url": .value})
            | tojson
          ')

          echo "has_refs=true" >> "$GITHUB_OUTPUT"
          echo "ref_count=${REF_COUNT}" >> "$GITHUB_OUTPUT"
          echo "matrix=${MATRIX}" >> "$GITHUB_OUTPUT"

          {
            echo "### ${CVE_ID}: ${REF_COUNT} reference(s) found"
            echo ""
            echo "Matrix prepared. Archiving will begin when downstream jobs are wired."
          } >> "$GITHUB_STEP_SUMMARY"
```

Key implementation requirements:
- Use `printf '%s'` rather than `echo` to pipe URLS into grep and jq — avoids a trailing newline adding a phantom empty URL
- The `jq -r` extraction uses `(.containers.cna.references // []) | .[].url` — the `// []` guard returns empty array (not null) when the path is absent, producing zero output lines instead of a jq error
- The `|| true` on the jq extraction prevents `set -e` from aborting when jq produces no output
- `grep -c .` counts non-empty lines; `|| echo "0"` handles the case where grep returns exit code 1 (no matches) — required because `set -e` is active
- The `exit 0` inside the REF_COUNT == 0 branch is deliberate — the job succeeds cleanly (no archive needed)
- `tojson` at the end of the jq pipeline produces a JSON string (not a pretty-printed object) as required for `echo "matrix=..." >> "$GITHUB_OUTPUT"`
- Do NOT use `echo "$URLS" | ...` for the matrix construction — use `printf '%s' "$URLS"` to avoid trailing newline creating an empty last element
